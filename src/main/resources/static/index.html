About the Project

This project provides a robust framework for evaluating RAG systems, focusing on key metrics that measure both retrieval quality and generation accuracy. Built with Spring AI and VectorStore, it offers a complete suite of tools for document storage, retrieval, and evaluation.

Agentic RAG Patterns Implementation

A working reference implementation using A2AJava framework for structured, deterministic agents

Overview

This implementation demonstrates Agentic Retrieval-Augmented Generation (Agentic RAG) using A2AJava, a structured Java agent framework. It follows design patterns that improve reliability, performance, and structure of retrieval workflows.

What the Paper Proposes

The Agentic RAG paper introduces a more structured and selective approach to information retrieval and reasoning. Unlike traditional RAG systems, Agentic RAG systems use intelligent agents to:

  • Decide whether a query needs retrieval at all
  • Choose which tools or actions to invoke
  • Execute deterministic steps when possible
  • Use LLMs or retrieval fallback only when necessary

Key Patterns

  • Agentic Routing
  • Tool Use
  • Reflection
  • Planning
  • Multi-Agent Workflows

What is A2AJava?

A2AJava is a Java framework for defining and invoking structured, deterministic agents. It is built on the Action-to-Action (A2A) protocol, enabling agents to declare actions, exchange JSON-compatible inputs and outputs, and collaborate using semantic memory and retrieval.

Multi-Language Support

Build agents in Java, Kotlin, Groovy, and Scala

Annotations

Use @Action, @Agent, and @Prompt annotations

Vector Integration

Built-in support for vector databases

LLM Fallback

Optional LLM/RAG fallback for open-ended inputs
